Technical Spec

GoHighLevel Workflow Blueprint

Complete automation spec for an AI receptionist system. Every trigger, condition, action, and webhook — mapped out and ready to build.

Prerequisites — Set These Up First

✓GHL Sub-Account created for client with Twilio integration active
✓Twilio Number provisioned and connected to GHL
✓GHL Calendar set up with client's availability windows
✓Custom Fields: call_type, service_needed, urgency_level, ai_transcript
✓Pipeline: "Inbound Leads" with stages: New → Qualified → Booked → Completed
✓Make/Zapier Webhook URL ready for AI processing handoff
01

Inbound Call → AI Agent Processing

Primary Workflow
Trigger

Inbound Phone Call

Fires when any call comes in to the Twilio number connected to the client's GHL sub-account.

trigger: "Call Status Changed"
filter: direction = "inbound"
filter: call_status = "completed"
// Fires after call ends so we have the recording URL
Action

Create/Update Contact

If caller is new, create contact. If existing, update with latest call data.

action: "Create/Update Contact"
phone: {{call.from}}
tag_add: "inbound-call", "ai-processed"
source: "Inbound Call"
Webhook

Send to Make.com for AI Processing

Sends call data to Make.com scenario which handles AI transcription, intent classification, and response generation.

action: "Outbound Webhook"
url: {{make_webhook_url}}
method: POST
body:
  contact_id: {{contact.id}}
  phone: {{contact.phone}}
  recording_url: {{call.recording_url}}
  call_duration: {{call.duration}}
  business_name: "{{client_business_name}}"

Make.com Scenario (built separately): Receives webhook → Transcribes recording via Whisper/Deepgram → Sends transcript to Claude/GPT with system prompt for intent classification → Returns: call_type, service_needed, urgency_level, customer_summary, should_book (boolean) → Sends result back to GHL via API.

Action

Update Contact with AI Results

Make.com webhook response updates the contact's custom fields with AI-extracted data.

action: "Update Contact Field"
call_type: {{webhook.call_type}} // emergency | routine | quote | info
service_needed: {{webhook.service_needed}}
urgency_level: {{webhook.urgency}} // high | medium | low
ai_transcript: {{webhook.summary}}
Action

Add to Pipeline

Creates an opportunity in the "Inbound Leads" pipeline at the appropriate stage.

action: "Add to Pipeline"
pipeline: "Inbound Leads"
stage: "Qualified"
opportunity_name: "{{contact.name}} - {{service_needed}}"
If urgency = "high"
Condition Branch

Route by Urgency Level

if urgency_level = "high"
  → Send Internal Alert SMS to owner
  → Send customer confirmation SMS (immediate)

if urgency_level = "medium"
  → Send customer confirmation SMS
  → Add to follow-up queue

if urgency_level = "low"
  → Send customer "we'll be in touch" SMS
  → Add to next-day follow-up queue
02

Missed Call → Instant Text-Back

Recovery Workflow
Trigger

Missed Call Detected

trigger: "Call Status Changed"
filter: call_status = "no-answer" OR "voicemail"
filter: direction = "inbound"
Action

Instant SMS Text-Back (within 60 seconds)

Critical for lead recovery. Most customers will text back before calling a competitor.

action: "Send SMS"
delay: 0 seconds
message: "Hi! Sorry we missed your call at {{business_name}}. We're available now — what can we help you with? Reply here or we'll call you right back."
Wait

Wait for Reply — 5 Minutes

wait_type: "reply"
timeout: 5 minutes
If reply received → AI SMS conversation
Webhook

Send Reply to Make.com for AI SMS Processing

AI reads their text, determines intent, and continues the conversation via SMS to book the job.

action: "Outbound Webhook"
url: {{make_sms_webhook_url}}
body:
  contact_id: {{contact.id}}
  message: {{contact.last_message}}
  context: "missed_call_recovery"
If no reply → Callback attempt
Action

Send Internal Alert for Manual Callback

action: "Internal Notification"
to: {{owner_phone}}
message: "Missed call from {{contact.phone}} — no text reply after 5 min. Call back ASAP."
03

Post-Service Follow-Up & Review Request

Retention Workflow
Trigger

Opportunity Moved to "Completed"

trigger: "Pipeline Stage Changed"
pipeline: "Inbound Leads"
stage: "Completed"
Wait

Wait 2 Hours After Service

wait: 2 hours
// Enough time for the customer to evaluate the work
Action

Send Thank-You + Review Request SMS

action: "Send SMS"
message: "Hi {{contact.first_name}}! Thanks for choosing {{business_name}}. We hope everything went well! If you have a moment, a Google review helps us a lot: {{google_review_link}} — Thank you!"
Wait

Wait 30 Days

wait: 30 days
Action

Reactivation Check-In

action: "Send SMS"
message: "Hi {{contact.first_name}}, just checking in from {{business_name}}! Everything still running smoothly? If you ever need us, just reply to this text. We're here for you."
tag_add: "reactivation-sent"
04

Stale Lead Re-Engagement

Nurture Workflow
Trigger

Contact Tag Added: "stale-lead"

Tag applied manually or via automation when a qualified lead hasn't booked within 7 days.

trigger: "Tag Added"
tag: "stale-lead"
Action

Re-Engagement SMS

action: "Send SMS"
message: "Hi {{contact.first_name}}, we noticed you reached out about {{service_needed}} but we never got to help. Still need assistance? We've got availability this week — just reply and we'll get you scheduled."
Wait

Wait 3 Days

wait: 3 days
Condition

Check if Reply Received

if replied = true → Route to AI SMS handler
if replied = false → Send final follow-up email
  subject: "Still need help with {{service_needed}}?"

Build Order: Start with Workflow 2 (Missed Call Text-Back) — it's the simplest, delivers immediate value, and proves the concept. Then build Workflow 1 (main AI pipeline). Save Workflows 3 and 4 for the second month once the core system is stable.
